Connexion

case LOGIN:

En cas de login, pas de vérification ici. On se contente de retirer à l’objet action ses champs type et password pour produire un objet user à utiliser comme nouvelle valeur de l’état (c’est-à-dire de currentUser).

En temps normal, on aurait ici une connexion au serveur, type Ajax ou JSON-P, en tous les cas asynchrone, et l’action renverrait par exemple une promesse, que le store Redux pourrait attendre au moyen d’un middleware type redux-thunk ou redux-promise. On peut aussi préférer recourir à une solution de type redux-actions.
